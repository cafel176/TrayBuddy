
TrayBuddy (托盘好友)
    ——你最好的托盘伴侣

1. 项目愿景 (Overview)
TrayBuddy 是一款基于 Tauri (Rust + Svelte) 的高性能桌面宠物应用。
它通过系统托盘常驻，平时作为屏幕边缘的轻量级圆形挂件，利用系统感知（音频、时间、硬件状态）提供情感化的交互、多媒体联动及待办提醒。

2. 核心视觉架构 (Visual Architecture)
2.1 悬浮圆形挂件 (Floating Avatar)
形态构成：窗口分为两部分——下半部分为固定的半圆形头像框，上半部分为动态人物实体。

窗口属性：
透明置顶：Webview 背景完全透明，窗口置顶。
点击穿透：通过 Rust 调用 set_ignore_cursor_events，确保鼠标点击宠物周围空白区域时能穿透到桌面底层。
位置控制：默认锚定在工作区（Workarea）右下角，支持边缘吸附，支持鼠标拖拽（Drag）。
点击反馈：单击触发随机“对话”弹出气泡，双击触发特定功能（如显示待办列表）。

动画选型 (Layered Animation)：
基础层：使用 Lottie 或 PNG 序列帧（低开销）。
进阶层：支持 Live2D (v3/v4) 或 Spine 渲染，实现视线追踪鼠标、物理摆动效果。
自适应降级：全屏游戏或开启省电模式时，窗口自动隐藏或停止渲染。

2.2 对话气泡系统 (Bubble System)
物理动效：弹出时具备弹性（Spring）动画。
富文本支持：集成简易 Markdown 解析，支持加粗、超链接、图片预览。
交互链：支持分支对话逻辑（A/B 选项），点击后触发新的 Rust 事件并反馈至 UI。
内容效果：支持打字机文本效果、同步播放语音。

3. 智能逻辑内核 (Intelligence Engine)
3.1 状态机与权重系统 (State Priority)
系统采用 权重优先级逻辑 决定当前宠物的表现：

优先级	状态类型	典型场景	表现行为
P0 (极高)	系统干扰	全屏游戏/办公、省电模式	自动隐身或沉睡
P1 (高)	交互触发	用户点击、拖拽	弹出对话、显示位移动画
P2 (中)	系统事件	待办到期、久坐提醒、天气剧变	强制弹出气泡提醒
P3 (低)	环境联动	音乐播放中、节日、特定时间点	切换为听歌、节日装扮状态
P4 (极低)	闲置逻辑	无外部输入	随机发呆、眨眼、随机对话

表现细节：
动画过渡：状态切换需包含 Transition Animation（如：从发呆切换到睡觉，需要一个“打哈欠”的过渡动作，而非瞬间切换）。

3.2 传感器与监听器 (Observers)
多媒体感知：通过 GlobalSystemMediaTransportControls (GSMTC) 监听系统音频，获取曲目信息并同步宠物摆动频率。
久坐提醒：检测持续运行时间超过 1 小时，提醒用户喝水/运动。

环境感知：
天气：每小时 API 轮询，本地缓存。联动动画（伞、围巾、冰棍）。
时间：毫秒级时钟，处理节日（圣诞帽）、生日、早晚问候。
季节：根据季节自动切换对应的动画资源库（如春季戴花，冬季戴帽）。

4. 扩展性与生态 (Extensibility)
4.1 模块化资源包 (.tbuddy)
文件结构：
text
/mod_name
├── manifest.json      # 元数据、剧本逻辑、触发规则
├── /assets            # 序列帧、Live2D 模型文件
└── /audio             # 语音及音效 (.mp3/ .ogg)

4.2 开发者接口 (Local API)
内置服务端：Rust 侧开启 localhost:5678 的轻量级 Server。
外部调用示例：
POST /command { "action": "talk", "text": "代码编译通过！", "emotion": "happy" }

5. 非功能性指标 (Non-Functional Requirements)
性能控制：
渲染挂起：当窗口被隐藏或不可见时，强制停止 Webview 的 RequestAnimationFrame。
内存目标：静默状态下内存占用控制在 40MB 以内。
数据安全：
本地优先：所有用户信息、待办事项、API Key 均加密存储在本地。
无感启动：静默添加到 Windows 注册表开机启动项。
国际化 (i18n)：
文本、语音库、时间格式根据系统 Locale 自动切换。
跨平台适配 (Cross-Platform Strategy)：
    核心逻辑解耦：建立独立的 `traybuddy-core` Rust Crate，封装状态机、剧本引擎、数值逻辑及多国语言解析。该内核不依赖任何特定平台的 GUI 库，确保其可作为独立库被 Android (JNI) 或 iOS (Swift) 项目调用。

6. 后续开发路线 (Roadmap)
MVP (最小可行产品)：托盘菜单 + 透明圆形窗口 + 基础发呆动画。
V1.1：接入系统媒体监听 + 对话气泡系统。
V1.2：待办提醒 + 本地持久化存储。
V2.0：Live2D 支持 + 创意工坊解压加载逻辑 + 本地 API 接口。
