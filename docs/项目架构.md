TrayBuddy 项目架构设计文档 (Comprehensive Architecture Design)
1. 整体架构概述
TrayBuddy 采用 Tauri v2 架构，将高性能的 Rust 后端与灵活的 Svelte 5 前端相结合。
为了满足跨平台和高性能需求，架构设计遵循“内核与表现分离”原则。

1.1 核心分层
- System Layer (Rust Core)：负责系统级交互（托盘、多媒体监听、定时器、文件 I/O、本地 API 服务）。
- Bridge Layer (Tauri IPC)：通过 Invoke 和 Event 实现前后端双向通信。
- UI Layer (Svelte Frontend)：负责渲染宠物动画、对话气泡、设置界面及状态展示。

2. 逻辑架构模块
2.1 后端模块 (src-tauri/src)
- tray_manager：管理系统托盘图标、原生菜单、开机自启动设置及托盘事件。
- window_manager：管理多窗口生命周期。
    - Avatar Window：半透明、置顶、忽略任务栏、支持点击穿透（set_ignore_cursor_events）。
    - Bubble Window：动态定位在 Avatar 窗口上方，支持弹性弹出效果。
- resource_manager：资产加载与索引系统。
    - 扫描 `mods/` 目录，解析各 mod 的 `manifest.json`。
    - 维护图像索引 (`assets/img.json`, `assets/sequence.json`)。
    - 维护多语言资源映射 (`audio/[lang]/`, `text/[lang]/`)。
- trigger_engine：异步任务中心。
    - TimeTrigger：处理整点提醒、闹钟、节日（如圣诞节）及生日逻辑。
    - MediaObserver：基于 Windows GSMTC 监听多媒体播放，获取曲目信息并同步节奏。
    - HealthNotifier：实现久坐提醒逻辑（持续运行 > 1h 触发）。
    - EnvironmentObserver：定时轮询天气 API 并联动季节系统。
- state_machine：维护宠物当前的核心状态，包含权重优先级计算与状态转移过渡（Transition）逻辑。
- storage：处理持久化数据（JSON/SQLite），支持加密存储用户信息、待办事项及 API Keys。
- api_server：集成轻量级 HTTP 服务（localhost:5678），暴露控制接口（如 `/command`）。

2.2 前端模块 (src/)
- lib/components：
    - Avatar.svelte：核心动画组件，支持 Lottie/序列帧/Live2D 渲染及视线追踪。
    - Bubble.svelte：富文本对话气泡，集成 Markdown 解析、打字机效果、分支选项及语音同步播放。
- lib/stores：
    - petStore：存储当前动作、心情、正在播放的音乐、i18n 语言包。
    - configStore：存储用户设置、偏好、权重配置。
- lib/animations：封装 GSAP 或 CSS Transitions，处理状态切换时的过渡动画（如打哈欠）。
- lib/i18n：处理多语言文本与本地化适配。

4. 关键技术方案设计
4.1 窗口定位与透明处理
- Avatar 窗口：在 tauri.conf.json 中配置 `transparent: true`。在 Rust 侧根据屏幕工作区（Workarea）动态计算右下角坐标，支持边缘吸附。
- 点击穿透：使用 `set_ignore_cursor_events` 确保非人物实体区域不拦截鼠标。

4.2 状态权重系统 (State Priority)
采用优先级抢占逻辑，确保核心体验：
| 优先级 | 状态类型 | 典型场景 | 表现行为 |
| :--- | :--- | :--- | :--- |
| P0 (极高) | 系统干扰 | 全屏游戏/办公 | 自动隐身或沉睡 (Render Suspend) |
| P1 (高) | 交互触发 | 用户点击、拖拽 | 弹出对话、弹性位移 |
| P2 (中) | 系统事件 | 待办到期、久坐提醒 | 弹出气泡提醒 |
| P3 (低) | 环境联动 | 音乐播放、天气变化 | 节奏摆动、装扮切换 |
| P4 (极低) | 闲置逻辑 | 无外部输入 | 随机发呆、眨眼 |

4.3 性能与资源管理
- 渲染挂起：当窗口不可见或处于 P0 状态时，强制停止 Webview 的 `requestAnimationFrame` 以降低 CPU/GPU 占用。
- 内存目标：通过 Rust 接管重负载逻辑，保持前端 Webview 轻量，目标静态内存 < 40MB。

4.4 模块化资源包 (.tbuddy)
资源加载器扫描 mods 目录，遵循以下层级结构解析 mod：
- 全局清单 (`manifest.json`)：定义 `id`, `version`, `author`, `important_actions` (如 `border`, `idle`) 及默认语音语言。
- 图像资产索引：
    - `assets/img.json`：定义静态或杂图资产。
    - `assets/sequence.json`：定义序列帧或网格动画资产，包含 `frame_time`, `frame_num_x/y` 等渲染参数。
- 多语言资源包：
    - 音频映射 (`audio/[lang]/speech.json`)：将动作/事件名称关联到具体的 `.wav` 或 `.mp3` 文件。
    - 文本信息 (`text/[lang]/`)：包含 `info.json` (角色多语言名) 和 `speech.json` (对话文本索引)。
- 路径映射：Rust 侧将上述 JSON 索引转换为内存中的 HashMap，并为前端提供动态资源路径解析。

4.5 本地 API 接口
开启 localhost:5678 服务，支持外部应用（如脚本或第三方插件）通过 POST 请求驱动宠物表现：
- 示例：`POST /command { "action": "talk", "text": "构建成功！", "emotion": "happy" }`

5. 开发里程碑 (Roadmap)
- Phase 1 (MVP)：实现透明窗口、托盘基础菜单、基础发呆动画、窗口点击穿透与边缘吸附。
- Phase 2 (Intelligence)：接入系统媒体监听 (GSMTC)、气泡系统（Markdown/打字机）、随机对话引擎、久坐提醒。
- Phase 3 (Personalization)：加密持久化存储、待办事项功能、天气与季节联动、多语言 (i18n) 支持。
- Phase 4 (Expansion)：Live2D 支持、.tbuddy 资源包加载逻辑、本地 API Server 完整实现。
