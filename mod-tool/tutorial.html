📖 Mod Editor 使用教程

本教程将指导你如何使用 TrayBuddy Mod Editor 创建和编辑角色 Mod 包。

简介

TrayBuddy Mod Editor 是一个可视化的 Mod 编辑工具，让你更直观地管理角色的各种数据。

📋
基础信息
配置 Mod ID、版本、作者、角色渲染参数、Mod 数据面板等
🎭
状态和触发
管理动作状态、对话分支、事件响应逻辑以及数据计数器
💬
气泡样式
自定义对话气泡样式，支持实时预览效果
🖼️
动画
管理序列帧动画和静态图片配置
📝
文本管理
多语言文本编辑，支持简易 Markdown
🔊
音频管理
管理多语言语音资源配置

环境要求

⚠️ 浏览器要求

请使用 Chrome 86+ 或 Edge 86+ 浏览器，其他浏览器暂不支持文件夹读写功能。

推荐运行方式：双击 mod-tool/打开-mod编辑器.bat 启动本地服务器（会自动打开浏览器），然后访问 http://127.0.0.1:4174/

快速开始

打开现有 Mod 文件夹

点击「打开 Mod」下拉菜单

在工具栏点击「📂 打开 Mod」按钮，选择「📂 打开文件夹」

选择 Mod 文件夹

在弹出的选择器中，选择一个 Mod 文件夹（如 ema）

授予访问权限

浏览器会询问是否允许读写该文件夹，点击「允许」

打开 .tbuddy 文件

.tbuddy 是 TrayBuddy 专用的 Mod 包格式，本质上是一个 ZIP 压缩包。点击「📦 打开 .tbuddy」选择文件后自动加载。

创建新 Mod

点击「➕ 新建 Mod」，填写 Mod ID、名称和作者，选择 Mod 类型（序列帧动画或 Live2D），系统会创建一个包含默认配置的新 Mod。

📋 基础信息配置

在「基础信息」标签页中配置 Mod 的核心参数。

Mod 信息

字段说明
Mod ID唯一标识符，只允许英文和数字
版本号Mod 版本
作者创作者名称
默认音频语言找不到对应语言音频时的备选
默认文本语言找不到对应语言文本时的备选

角色配置

配置角色渲染相关参数（manifest.character）。

字段说明
z_offset角色画布的渲染层级偏移
canvas_fit_preference角色 Canvas 显示适配：short(短边优先)/long(长边优先)/legacy(旧版按高度缩放)

边框配置

配置边框开关、边框动画与层级（manifest.border）。

字段说明
enable是否启用边框
anima边框使用的动画名称（从动画资源中选择）
z_offset边框画布的渲染层级偏移

Mod 数据面板

可在角色窗口左上角显示自定义数据（如好感度）。配置 show_mod_data_panel 开关和 mod_data_default_int 默认值。

全局键盘监听

配置 global_keyboard 为 true 后，即使动画窗口未获得焦点，也能触发 keydown 事件。适用于需要实时响应键盘输入的桌面宠物模型。

全局鼠标监听

配置 global_mouse 为 true 后，鼠标点击任意位置都可触发 global_click / global_right_click 事件，无需点击在角色身上。

预览图与图标

预览图支持 PNG/JPG/JPEG/WebP 格式，图标支持 ICO 格式。

🎭 状态和触发

重要状态

系统内置的关键状态：

状态名称持久说明
idle✅默认空闲状态，必须存在
silence✅静默/免打扰状态
silence_start❌进入免打扰时播放一次，然后跳转到 silence
silence_end❌退出免打扰时播放一次，然后跳转到 idle
dragging✅拖动中状态（持续）
drag_start❌开始拖动时播放一次，然后跳转到 dragging
drag_end❌结束拖动时播放一次，然后跳转到 idle（或后续由系统进入 music）
music✅音乐播放状态
music_start❌开始听歌时播放一次，然后跳转到 music
music_end❌停止听歌时播放一次，然后跳转到 idle
birthday❌生日事件状态（通常播放一次）
firstday❌首次启动事件状态（通常播放一次）

状态字段

字段说明
状态名称唯一标识符
是否持久持久状态循环播放
关联动画要播放的动画
关联音频音频资源名称
关联文本显示为气泡对话
优先级数值越大优先级越高
下一状态播放完成后跳转
触发间隔定时触发检查间隔（秒），0 表示不启用
触发概率每次检查的触发概率（编辑器以百分比显示，保存到 manifest 为 0-1）
触发计数起点仅当当前 ModData.value 落在 [起点, 终点] 区间内时，该状态才允许触发
触发计数终点仅当当前 ModData.value 落在 [起点, 终点] 区间内时，该状态才允许触发
气温触发起点仅当当前 environment.temperature (°C) 落在 [起点, 终点] 区间内时，该状态才允许触发
气温触发终点仅当当前 environment.temperature (°C) 落在 [起点, 终点] 区间内时，该状态才允许触发
启动时长触发仅当当前距离程序启动已过分钟数 >= trigger_uptime 时，该状态才允许触发（分钟；0 表示不限制）
触发天气仅当当前 environment.condition（或 condition_code）与 trigger_weather 任意一项精确匹配时，该状态才允许触发（数组）
可触发子状态在编辑器内以“状态 + 权重”列表配置（权重越大越容易被选中）

对话分支

对话分支允许用户通过点击选项来决定对话走向。每个分支包含显示文本和跳转目标状态。

“显示气泡”(branch_show_bubble) 用于控制分支选项是否以气泡形式展示；该选项位于「对话分支」折叠区内。

触发器配置

触发器定义事件与状态的关联。常用事件（支持 keydown:<Key>）：

事件名触发时机
click用户点击角色
right_click用户右键点击角色
global_click全局左键点击（需开启全局鼠标监听）
global_right_click全局右键点击（需开启全局鼠标监听）
global_keydown全局键盘按下（任意键，需开启全局键盘监听）
login程序启动
drag_start开始拖动窗口
drag_end结束拖动窗口
keydown:<Key>按键触发（例如：keydown:Space）
birthday生日事件
firstday首次启动事件
login_silence启动后进入静默/无声状态
music_start音乐开始播放
music_end音乐停止播放
work工作事件

允许重复触发

每个触发状态组都有一个"允许重复触发"(allow_repeat)选项。关闭后，同一状态不会连续触发两次（仅当该组状态数大于1时生效）。

数据计数器

状态激活时对 Mod 数据执行操作：set(设置)、add(加)、sub(减)、mul(乘)、div(除)。

你也可以在编辑器的「状态限制」中，通过 trigger_counter_start / trigger_counter_end 限制该状态在特定 ModData.value 区间内才允许触发。

同样地，你也可以通过 trigger_temp_start / trigger_temp_end 限制该状态仅在气温（environment.temperature，单位 °C）处于区间内才允许触发。

此外，你也可以通过 trigger_uptime 限制该状态仅在程序启动后已运行达到一定分钟数时才允许触发（0 表示不限制）。

此外，你也可以通过 trigger_weather 限制该状态仅在天气满足条件时才允许触发：若填纯数字则匹配 weatherCode（condition_code），否则匹配天气描述（condition）。

💬 气泡样式配置

可视化配置对话气泡外观，支持实时预览。

  • 气泡主体：背景、边框、圆角、阴影、背景滤镜
  • 文字样式：颜色、字体大小、行高、字体
  • 尾巴样式：大小、颜色、阴影
  • 装饰文字：上方/下方装饰字符
  • 分支按钮：正常/悬浮/点击状态样式

🖼️ 动画管理

动画资源分为序列帧动画(sequence.json)和杂图(img.json)两类。

字段说明
名称资源名称，供状态引用
图片路径相对于 asset/ 目录
是否序列帧是否为多帧动画
帧间隔每帧显示时间(秒)
帧尺寸单帧宽高(像素)
帧数横向/纵向帧数量
需要反向播放开启后会以乒乓方式播放（默认关闭，可在高级选项中设置）
原始反向图片序列本身为反向时可开启，用于纠正方向
偏移X/Y 偏移（像素），用于微调渲染位置
⚠️ 注意

动画图片需要手动放置到 asset/sequence/ 或 asset/img/ 目录。

🎭 Live2D 动画

Live2D Mod 使用 Live2D Cubism 模型代替序列帧动画。创建新 Mod 时选择 "Live2D" 类型即可。

模型配置

在「动画」标签页中配置 Live2D 模型的基本参数。

字段说明
name模型显示名称
base_dir模型资源根目录，相对于 Mod 目录
model_json模型描述文件名（通常为 .model3.json）
textures_dir贴图目录
motions_dir动作文件目录
expressions_dir表情文件目录
physics_json物理运算配置文件
pose_json姿势配置文件
eye_blink是否启用自动眨眼
lip_sync是否启用口型同步

动作配置

管理 Live2D 模型的动作（Motion）列表，每个动作对应一个 .motion3.json 文件。

字段说明
name动作名称，供状态映射引用
file动作文件路径（相对于 motions_dir）
group动作分组（如 idle、tap 等）
priority播放优先级
fade_in_ms / fade_out_ms淡入/淡出时间（毫秒）
loop是否循环播放

表情配置

管理 Live2D 模型的表情列表，每个表情对应一个 .exp3.json 文件。

字段说明
name表情名称
file表情文件路径（相对于 expressions_dir）

状态映射

状态映射定义了每个应用状态应使用的 Live2D 动作和表情。这些条目会在「状态和触发」页的动画下拉中可选。

字段说明
state状态名称（与 manifest 中的状态名对应）
motion关联的动作名称
expression关联的表情名称（可选）
scale模型缩放比例
offset_x / offset_y位置偏移量

从文件同步

点击「从文件同步」按钮，可从 Live2D 模型目录中自动读取配置：

  • 自动检测模型：若 base_dir 为空则自动使用 asset/live2d/，自动扫描 .model3.json 文件
  • 自动填充配置：从 model3.json 的 FileReferences 自动推断 name、model_json、physics_json、pose_json、textures_dir、motions_dir、expressions_dir、eye_blink、lip_sync
  • 自动导入动作/表情：从 model3.json 解析完整的动作和表情列表
  • 自动生成状态映射：为每个动作创建对应的状态-动画映射条目（已有映射会保留用户自定义的参数）

新增同名状态

每个状态映射条目右侧的「新增同名状态」按钮可快速在 manifest 中创建对应的状态，并自动联动：

  • 关联动画：自动设为当前映射名称
  • 关联文本：若存在同名的对话文本条目，自动关联
  • 关联音频：若存在同名的音频条目，自动关联
💡 提示

Live2D 模型文件需要手动放置到对应目录。建议参考 mods/mao_pro 的结构组织文件。

💡 导出说明

导出 Live2D Mod 时，asset/live2d/ 目录下的所有文件（包括 .model3.json、.motion3.json、.exp3.json、.physics3.json 等）都会被完整打包。

📝 多语言文本

配置多语言的角色信息和对话内容。

角色信息 (info.json)

语言ID、语言名称、角色名称、角色描述

对话文本 (speech.json)

名称、内容(支持Markdown)、持续时间(秒)

支持占位符：{nickname}（昵称）、{days_used}（使用天数）、{usage_hours}/{total_usage_hours}（总使用时长，小时）、{uptime}（本次启动已运行时长，HH:MM:SS）

新增同名状态

每条对话文本右侧有「新增同名状态」按钮，点击后可快速在 manifest 中创建同名状态，并自动联动：

  • 关联文本：自动设为当前文本名称
  • 关联动画：若 Live2D 状态映射中存在同名条目，自动关联
  • 关联音频：若音频资源中存在同名条目，自动关联

Markdown 语法

  • **文本** → 加粗
  • [文本](链接) → 超链接
  • \n → 换行

🔊 多语言音频

配置多语言的语音资源。音频文件支持 .wav 格式，需要手动放置到对应目录。

💾 保存与导出

保存到文件夹

点击「💾 保存」（或「📦 导出」→「📂 到文件夹」），选择目标文件夹后写入。当前版本每次保存都会弹出文件夹选择器。

导出为 .tbuddy

点击「📦 导出」→「📦 到 .tbuddy」生成压缩包（包内会以 Mod ID 作为根目录）。

保存/导出的文件

  • manifest.json
  • bubble_style.json（开启气泡样式时生成）
  • asset/sequence.json, asset/img.json（序列帧 Mod）
  • asset/live2d.json（Live2D Mod）
  • asset/live2d/ 下的所有文件（.model3.json、.moc3、贴图、.motion3.json、.exp3.json 等）将完整打包（Live2D Mod）
  • text/[lang]/info.json, text/[lang]/speech.json（开启文本对话时生成）
  • audio/[lang]/speech.json（开启音频对话时生成）
  • preview.(png/jpg/jpeg/webp), icon.ico

📁 Mod 目录结构

序列帧 Mod (mod_type: "sequence")

mods/my_mod/
├── manifest.json            # Mod 主配置文件
├── preview.png              # 预览图
├── icon.ico                 # 图标
├── bubble_style.json        # [可选] 气泡样式
├── asset/
│   ├── img.json, sequence.json
│   ├── img/, sequence/
├── audio/[lang]/
│   ├── speech.json, speech/
└── text/[lang]/
    ├── info.json, speech.json

Live2D Mod (mod_type: "live2d")

mods/my_mod/
├── manifest.json            # Mod 主配置文件
├── preview.png              # 预览图
├── icon.ico                 # 图标
├── bubble_style.json        # [可选] 气泡样式
├── asset/
│   ├── live2d.json          # Live2D 配置（动作/表情/状态映射）
│   └── live2d/              # Live2D 模型资源目录
│       ├── *.model3.json    # 模型描述
│       ├── *.moc3           # 模型二进制
│       ├── *.physics3.json  # [可选] 物理配置
│       ├── textures/        # 贴图
│       ├── motions/         # 动作 (.motion3.json)
│       └── expressions/     # 表情 (.exp3.json)
├── audio/[lang]/
│   ├── speech.json, speech/
└── text/[lang]/
    ├── info.json, speech.json

❓ 常见问题

Q: 为什么无法打开文件夹？

A: 请使用 Chrome 86+ 或 Edge 86+ 浏览器。

Q: 动画不显示？

A: 检查图片是否在正确目录，路径和帧配置是否正确。

Q: 如何添加新语言？

A: 在「文本管理」或「音频管理」中点击「➕ 添加语言」。

Q: 预览图支持哪些格式？

A: 支持 PNG、JPG、JPEG、WebP 四种格式。

Q: 为什么提示 i18n 资源加载失败 / file:// 被拦截？

A: 请不要用 file:// 直接打开页面。双击 mod-tool/打开-mod编辑器.bat 启动本地服务器，再用浏览器访问 http://127.0.0.1:4174/。
