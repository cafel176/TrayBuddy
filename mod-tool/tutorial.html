📖 Mod Editor 使用教程

本教程将指导你如何使用 TrayBuddy Mod Editor 创建和编辑角色 Mod 包（文件夹或 .tbuddy）。

简介

TrayBuddy Mod Editor 是一个可视化的 Mod 编辑工具，采用顶部标签页设计，让你更直观地管理角色的各种数据。

📋
基础信息
配置 Mod ID、版本、作者、角色渲染参数等
🎭
状态和触发
管理动作状态、对话分支、事件响应逻辑以及数据计数器（Mod Data Counter）
🧼
气泡样式
自定义对话气泡样式，支持背景滤镜、装饰文字和阴影效果
🖼️
动画
管理序列帧动画和静态图片配置
💬
文本管理
多语言文本编辑，支持简易 Markdown
🔊
音频管理
管理多语言语音资源配置

快速开始

打开现有 Mod

现在支持两种方式打开 Mod：

  • 打开文件夹：直接选择电脑上的 Mod 目录。
  • 打开 .tbuddy：选择打包好的 .tbuddy 文件，编辑器会自动解压读取。

什么是 .tbuddy？

.tbuddy 是 TrayBuddy 专用的 Mod 包格式，本质上是一个包含 Mod 完整目录结构的 ZIP 压缩包。你可以通过编辑器的「导出」功能生成它。

🎭 状态和触发

我们将状态管理和触发器配置整合到了同一个页面，左侧管理状态，右侧管理事件响应。

💡 提示

现在可以在「可触发子状态」中设置权重，格式为 state_name:weight，例如 happy:2, normal:1 表示 happy 触发概率是 normal 的两倍。

数据计数器 (Mod Data Counter)

你可以在状态激活时对角色的自定义变量进行操作，支持 set、add、sub、mul、div。这些变量可以被程序其他部分引用（如显示好感度）。

🧼 气泡样式配置

在「气泡样式」标签页中，你可以通过可视化表单修改 bubble_style.json。这些设置会直接影响桌面角色的对话气泡外观。

  • 背景：支持颜色代码或 CSS 渐变。
  • 背景滤镜：支持 CSS backdrop-filter，如 blur(8px) 实现毛玻璃效果。
  • 装饰文字：可以在气泡上方或下方显示固定的装饰字符。
  • 尾巴阴影：为气泡小尾巴添加阴影，增强立体感。
  • 分支按钮：支持配置普通、悬浮、点击（Active）三种状态的颜色。

快速开始

打开现有 Mod

点击「打开 Mod」按钮

在工具栏右侧点击 按钮

选择 Mod 文件夹

在弹出的文件夹选择器中，导航到 mods 目录，选择一个 Mod 文件夹（如 ema）

授予访问权限

浏览器会询问是否允许读写该文件夹，点击「允许」

开始编辑

Mod 加载成功后，所有配置会自动填充到对应的编辑面板中

创建新 Mod

点击「新建 Mod」按钮

在工具栏右侧点击 按钮

填写基本信息

在弹窗中输入：

  • Mod ID：唯一标识符，只能包含英文、数字和下划线（如 my_character）
  • Mod 名称：角色的显示名称
  • 作者：你的名字
配置内容

系统会创建一个包含默认配置的新 Mod，你可以在各个面板中进行编辑

保存 Mod

编辑完成后，点击「保存」并选择保存位置（建议保存到 mods 目录下）

📋 基本信息配置

在「基本信息」面板中配置 Mod 的核心参数。

Mod 信息

字段 说明 示例
Mod ID 唯一标识符，用于程序识别 ema
版本号 Mod 版本，用于更新检测 1.0.0
作者 创作者名称 Cafel

语言设置

字段 说明 示例
默认音频语言 找不到对应语言音频时使用的备选语言 jp
默认文本语言 找不到对应语言文本时使用的备选语言 zh

角色配置

  • Z轴偏移：角色的渲染层级，数值越大越靠前

边框配置

  • 边框动画：选择用于边框的动画资源名称
  • 启用边框：是否显示角色边框
  • Z轴偏移：边框的渲染层级
💡 提示

预览图建议使用 PNG 格式，尺寸推荐 512×512 像素。

🎭 状态管理

状态（State）定义了角色在特定情况下的表现，包括播放的动画、音频和显示的文本。

重要状态 (Idle)

idle 是角色的默认待机状态，当没有其他状态激活时，角色会回到此状态。这个状态必须存在且为持久状态。

普通状态

点击「➕ 添加状态」可以创建新的状态。每个状态可以设置不同的动画、音频和对话内容。

状态字段说明

字段 类型 说明
状态名称 文本 唯一标识符，用于触发器和分支跳转
是否持久 开关 持久状态会循环播放，非持久状态播放一次后结束
关联动画 下拉选择 从动画资源中选择要播放的动画
关联音频 文本 填写音频资源名称（需要在音频管理中配置）
关联文本 文本 填写文本名称（需要在文本管理中配置），将显示为气泡对话
优先级 数字 数值越大优先级越高，高优先级状态会打断低优先级状态
日期范围 MM-DD 该状态生效的日期范围（可选）
时间范围 HH:MM 该状态生效的时间范围（可选）
下一状态 文本 播放完成后自动跳转到的状态名称
可触发子状态 文本列表 处于该状态时可能随机触发的状态，逗号分隔
触发间隔 秒 随机触发的最小间隔时间
触发概率 0-1 每次检测时触发的概率

对话分支

对话分支允许用户通过点击选项来决定对话走向。气泡显示文本后会出现分支按钮供用户选择。

✅ 分支示例

状态 hello1_1 设置了两个分支：

  • 「好的！」→ 跳转到 hello1_2
  • 「再说一次」→ 跳转到 hello1_1（重复当前状态）

添加分支的步骤：

  1. 在状态编辑弹窗底部找到「对话分支」区域
  2. 点击「➕ 添加分支」按钮
  3. 填写选项文本（用户看到的按钮文字）
  4. 填写跳转状态（点击后切换到的状态名称）

⚡ 触发器配置

触发器定义了当特定事件发生时，角色可能切换到哪些状态。

预设事件

事件名 触发时机
click 用户点击角色时
login 程序启动或用户登录时
music_start 检测到系统音乐播放时
music_end 检测到系统音乐停止时

配置方法

  1. 点击「➕ 添加触发器」或编辑现有触发器
  2. 输入可触发的状态列表（逗号分隔多个状态名）
  3. 当事件触发时，系统会从列表中随机选择一个状态执行
💡 示例

login 触发器配置了 morning, noon, evening, night，程序会根据当前时间自动选择合适的问候状态。

🖼️ 动画资源

动画资源定义了如何解析和播放图片。分为两类：

  • 序列帧动画 (sequence.json)：复杂的动作动画，如眨眼、挥手
  • 杂图资源 (img.json)：简单的图片，如边框

动画配置字段

字段 说明
名称 资源名称，供状态引用
图片路径 相对于 asset/ 目录的路径
是否序列帧 是否为多帧动画
原始反向 图片中的帧是否是反向排列的
需要反向播放 播放完成后是否反向播放回起始帧
帧间隔 (秒) 每帧显示的时间
帧宽度/高度 单帧的尺寸（像素）
横向/纵向帧数 图片中横向和纵向排列的帧数量
X/Y偏移 渲染时的位置偏移（像素）
⚠️ 注意

动画图片需要手动放置到 asset/sequence/ 或 asset/img/ 目录中，编辑器只负责配置元数据。

💬 文本管理

文本管理用于配置多语言的角色信息和对话内容。

角色信息 (info.json)

  • 语言ID：语言的标识符（如 zh、jp、en）
  • 语言名称：语言的显示名称（如「简体中文」「日本語」）
  • 角色名称：角色在该语言下的名字
  • 角色描述：角色的介绍文字

对话文本 (speech.json)

对话文本与状态中的 text 字段关联。当状态触发时，会根据文本名称查找并显示对应的对话内容。

Markdown 语法支持

对话文本支持简易 Markdown 语法：

语法 效果 示例
**文本** 加粗 **你好！** → 你好！
[文本](链接) 超链接 [点我](https://example.com)
\n 换行 第一行\n第二行 
{
  "name": "greeting",
  "text": "**你好呀！** 我是你的桌面伙伴～\n有什么可以帮助你的吗？"
}

🔊 音频管理

音频管理用于配置多语言的语音资源。

配置字段

  • 名称：音频的标识符，供状态的 audio 字段引用
  • 文件路径：相对于 audio/ 目录的路径（如 jp/speech/morning.wav）
💡 提示

音频文件支持 .wav 格式。文件需要手动放置到对应目录中。

💾 保存与导出

保存 Mod

点击工具栏的 按钮：

  • 如果是打开的现有 Mod，会直接覆盖保存到原文件夹
  • 如果是新建的 Mod，会弹出文件夹选择器让你选择保存位置

保存的文件

保存操作会写入以下文件：

  • manifest.json - 主配置文件
  • asset/sequence.json - 序列帧动画配置
  • asset/img.json - 杂图配置
  • text/[lang]/info.json - 各语言角色信息
  • text/[lang]/speech.json - 各语言对话文本
  • audio/[lang]/speech.json - 各语言音频配置
  • preview.png - 预览图（如果有上传）
⚠️ 注意

图片文件（动画图、音频文件）不会被编辑器自动复制，需要手动放置到对应目录。

📁 Mod 目录结构

一个完整的 Mod 文件夹结构如下：

mods/my_mod/
├── manifest.json            # Mod 主配置文件
├── preview.png              # 预览图
├── asset/                   # 动画图像资源
│   ├── img.json             # 杂图索引
│   ├── sequence.json        # 序列帧动画索引
│   ├── img/                 # 杂图存放目录
│   │   └── border.png
│   └── sequence/            # 序列帧动画图片
│       ├── wink.png
│       └── wave.png
├── audio/                   # 音频资源
│   └── jp/                  # 日语音频
│       ├── speech.json      # 音频索引
│       └── speech/          # 音频文件
│           └── morning.wav
└── text/                    # 文本资源
    ├── zh/                  # 中文
    │   ├── info.json        # 角色信息
    │   └── speech.json      # 对话文本
    └── jp/                  # 日语
        ├── info.json
        └── speech.json

❓ 常见问题

Q: 为什么无法打开文件夹？

A: 请确保使用 Chrome 86+ 或 Edge 86+ 浏览器，其他浏览器不支持 File System Access API。

Q: 保存后动画不显示？

A: 检查以下几点：

  1. 动画图片是否放在了正确的目录（asset/sequence/ 或 asset/img/）
  2. 动画配置中的图片路径是否正确
  3. 帧数和帧尺寸设置是否与实际图片匹配

Q: 如何添加新语言？

A: 在「文本管理」或「音频管理」页面，点击语言标签旁边的「➕ 添加语言」按钮，输入语言 ID（如 en）即可。

Q: 对话气泡不显示？

A: 确保：

  1. 状态的 text 字段填写了文本名称
  2. 在「文本管理」中添加了对应名称的对话文本
  3. 选择了正确的语言

Q: 标题栏显示 * 是什么意思？

A: 星号表示有未保存的更改，请及时点击「保存」按钮。
