📖 Mod Editor 使用教程

本教程将指导你如何使用 TrayBuddy Mod Editor 创建和编辑角色 Mod 包。

简介

TrayBuddy Mod Editor 是一个可视化的 Mod 编辑工具，让你更直观地管理角色的各种数据。

📋
基础信息
配置 Mod ID、版本、作者、角色渲染参数、Mod 数据面板等
🎭
状态和触发
管理动作状态、对话分支、事件响应逻辑以及数据计数器
💬
气泡样式
自定义对话气泡样式，支持实时预览效果
🤖
AI 工具
配置 AI 截图识别工具，自动识别游戏画面触发状态
🖼️
动画
管理序列帧动画和静态图片配置
📝
文本管理
多语言文本编辑，支持简易 Markdown
🔊
音频管理
管理多语言语音资源配置

环境要求

⚠️ 浏览器要求

请使用 Chrome 86+ 或 Edge 86+ 浏览器，其他浏览器暂不支持文件夹读写功能。

推荐运行方式：双击 mod-tool/打开-mod编辑器.bat 启动本地服务器（会自动打开浏览器），然后访问 http://127.0.0.1:4174/

快速开始

打开现有 Mod 文件夹

点击「打开 Mod」下拉菜单

在工具栏点击「📂 打开 Mod」按钮，选择「📂 打开文件夹」

选择 Mod 文件夹

在弹出的选择器中，选择一个 Mod 文件夹（如 ema）

授予访问权限

浏览器会询问是否允许读写该文件夹，点击「允许」

打开 .tbuddy 文件

.tbuddy 是 TrayBuddy 专用的 Mod 包格式，本质上是一个 ZIP 压缩包。点击「📦 打开 .tbuddy」选择文件后自动加载。

创建新 Mod

点击「➕ 新建 Mod」，填写 Mod ID、名称和作者，选择 Mod 类型（序列帧动画、Live2D、PngRemix 或 3D 模型），系统会创建一个包含默认配置的新 Mod。

在「基础信息」标签页中配置 Mod 的核心参数。

Mod 信息

作者创作者名称
默认音频语言找不到对应语言音频时的备选

角色配置

配置角色渲染相关参数（manifest.character）。

字段说明

canvas_fit_preference角色 Canvas 显示适配：short(短边优先)/long(长边优先)/legacy(旧版按高度缩放)

字段说明

Mod 数据面板

可在角色窗口左上角显示自定义数据（如好感度）。配置 show_mod_data_panel 开关和 mod_data_default_int 默认值。

贴图降采样

通过 manifest.enable_texture_downsample 控制是否允许贴图在解码阶段降采样/封顶（对 Live2D/PngRemix 生效）。开启可降低内存占用，关闭可保留原始清晰度。

通过 manifest.texture_downsample_start_dim 设置"开始降采样的贴图尺寸阈值"（像素；最长边）。0 表示不限制；>0 表示仅当贴图尺寸达到阈值时才允许触发降采样。

全局键盘监听

配置 global_keyboard 为 true 后，即使动画窗口未获得焦点，也能触发 keydown 事件。适用于需要实时响应键盘输入的桌面宠物模型。

全局鼠标监听

配置 global_mouse 为 true 后，鼠标点击任意位置都可触发 global_click / global_right_click 事件，无需点击在角色身上。

预览图与图标

预览图支持 PNG/JPG/JPEG/WebP 格式，图标支持 ICO 格式。

对话分支

对话分支允许用户通过点击选项来决定对话走向。每个分支包含显示文本和跳转目标状态。

"显示气泡"(branch_show_bubble) 用于控制分支选项是否以气泡形式展示；该选项位于「对话分支」折叠区内。

click_up左键松开（窗口内）
right_click用户右键点击角色
right_click_up右键松开（窗口内）
global_click全局左键点击（需开启全局鼠标监听）
global_click_up全局左键松开（需开启全局鼠标监听）
global_right_click全局右键点击（需开启全局鼠标监听）
global_right_click_up全局右键松开（需开启全局鼠标监听）
global_keydown全局键盘按下（任意键，需开启全局键盘监听）
global_keyup全局键盘松开（任意键，需开启全局键盘监听）

允许重复触发

每个触发状态组都有一个"允许重复触发"(allow_repeat)选项。关闭后，同一状态不会连续触发两次（仅当该组状态数大于1时生效）。

数据计数器

状态激活时对 Mod 数据执行操作：set(设置)、add(加)、sub(减)、mul(乘)、div(除)。

你也可以在编辑器的「状态限制」中，通过 trigger_counter_start / trigger_counter_end 限制该状态在特定 ModData.value 区间内才允许触发。

同样地，你也可以通过 trigger_temp_start / trigger_temp_end 限制该状态仅在气温（environment.temperature，单位 °C）处于区间内才允许触发。

此外，你也可以通过 trigger_uptime 限制该状态仅在程序启动后已运行达到一定分钟数时才允许触发（0 表示不限制）。

此外，你也可以通过 trigger_weather 限制该状态仅在天气满足条件时才允许触发：若填纯数字则匹配 weatherCode（condition_code），否则匹配天气描述（condition）。

列表操作

状态列表和触发器列表提供多种便捷操作：

  • 筛选器：支持按名称、关联动画、关联音频、关联文本等字段过滤列表内容
  • 复制/粘贴：可将状态或触发器复制到剪切板，在不同 Mod 之间快速迁移数据
  • 拖拽排序：拖拽列表条目可重新排列顺序（使用筛选器时需先清空条件）
  • 全部删除：一键清空所有普通状态或所有触发器

💬 气泡样式配置

可视化配置对话气泡外观，支持实时预览。

  • 气泡主体：背景、边框、圆角、阴影、背景滤镜
  • 文字样式：颜色、字体大小、行高、字体
  • 尾巴样式：大小、颜色、阴影
  • 装饰文字：上方/下方装饰字符
  • 分支按钮：正常/悬浮/点击状态样式

🎭 Live2D 动画

模型配置

lip_sync是否启用口型同步

动作配置

管理 Live2D 模型的动作（Motion）列表，每个动作对应一个 .motion3.json 文件。

字段说明

表情配置

管理 Live2D 模型的表情列表，每个表情对应一个 .exp3.json 文件。

字段说明

file表情文件路径（相对于 expressions_dir）

状态映射

状态映射定义了每个应用状态应使用的 Live2D 动作和表情。这些条目会在「状态和触发」页的动画下拉中可选。

字段说明

背景图层

背景图层用于在 Live2D 模型的下方或上方渲染额外的图片层，适用于静态背景、按键叠加高亮等场景。

字段说明
name图层名称（标识用）
file图片文件路径（相对于 base_dir）

offset_x / offset_y位置偏移量（像素）
events关联事件名列表，任一事件触发时显示；为空则常驻显示
audio触发时播放的音效名称（空字符串表示不播放）
dir图片所在目录（可选，用于分组筛选）

从文件同步

点击「从文件同步」按钮，可从 Live2D 模型目录中自动读取配置：

  • 自动检测模型：若 base_dir 为空则自动使用 asset/live2d/，自动扫描 .model3.json 文件
  • 自动填充配置：从 model3.json 的 FileReferences 自动推断 name、model_json、physics_json、pose_json、textures_dir、motions_dir、expressions_dir、eye_blink、lip_sync
  • 自动导入动作/表情：从 model3.json 解析完整的动作和表情列表
  • 自动生成状态映射：为每个动作创建对应的状态-动画映射条目（已有映射会保留用户自定义的参数）

从文件生成输入事件

仅适用于 Live2D Mod。点击「⌨️ 从文件生成输入事件」按钮，可从 cdi3.json（DisplayInfo）中识别键盘/鼠标相关参数，自动批量生成对应的状态和触发器（含 keydown/keyup 事件），并自动开启全局键盘/鼠标监听。适用于需要实时响应键盘输入的桌面宠物模型（如 BongoCat）。

模型参数浏览器

Live2D 动画标签页内的可折叠面板。点击「从文件读取」可加载并展示 cdi3.json 中定义的所有模型参数，按分组折叠显示，支持组合参数识别。方便查看可用的参数 ID，用于配置 live2d_params 覆写。

新增同名状态

每个状态映射条目右侧的「新增同名状态」按钮可快速在 manifest 中创建对应的状态，并自动联动：

  • 关联动画：自动设为当前映射名称
  • 关联文本：若存在同名的对话文本条目，自动关联
  • 关联音频：若存在同名的音频条目，自动关联
💡 提示

Live2D 模型文件需要手动放置到对应目录。建议参考 mods/mao_pro 的结构组织文件。

导入文件夹

点击「📂 导入文件夹」按钮，可选择一个外部文件夹，将其内容递归复制到 Mod 的 asset/live2d/ 目录下。适合从外部位置批量导入 Live2D 模型资源。

💡 导出说明

导出 Live2D Mod 时，asset/live2d/ 目录下的所有文件（包括 .model3.json、.motion3.json、.exp3.json、.physics3.json 等）都会被完整打包。

offset_x / offset_y位置偏移量

PngRemix 参数覆写

在状态编辑面板中，PngRemix 类型的 Mod 可使用 pngremix_params 在进入状态时切换表情或触发动作。每条参数包含 type（expression 或 motion）和 name。

导入文件

点击「📂 导入文件」按钮，可选择一个 .pngRemix 文件并自动复制到 Mod 的 asset/ 目录下，同时更新 pngremix_file 路径配置。

从文件同步

点击同步按钮可从 .pngRemix 文件中自动读取配置：

  • 从文件同步配置：从 .pngRemix 文件读取 blink_speed、blink_chance 等模型特性参数，自动填充到配置中
  • 从文件同步资产：从 .pngRemix 文件扫描所有 state/hotkey 信息，自动生成表情和动作列表

新增同名状态

每个状态映射条目右侧的「新增同名状态」按钮可快速在 manifest 中创建对应的状态，并自动联动：

  • 关联动画：自动设为当前映射名称
  • 关联文本：若存在同名的对话文本条目，自动关联
  • 关联音频：若存在同名的音频条目，自动关联
💡 提示

.pngRemix 模型文件需要使用 PNGTuber Remix 编辑器创建。将 .pngRemix 文件放置到 asset/ 目录下，然后在编辑器中配置路径。

🧊 3D 动画

3D Mod 使用 VRM 或 PMX 模型文件进行 3D 渲染。创建新 Mod 时选择 "3D 模型" 类型即可。支持 VRM + VRMA 和 PMX + VMD 两种模型/动画组合。

模型配置

在「动画」标签页中配置 3D 模型的基本参数。

字段说明
name模型显示名称
type模型类型：vrm（VRM 格式）或 pmx（PMX 格式）
file模型文件路径（相对于 Mod 根目录，如 asset/3d/model.vrm）
scale整体缩放系数（默认 1）
offset_x / offset_y位置偏移量
texture_base_dir纹理文件基础目录（PMX 模型使用，可选）
animation_base_dir动画文件基础目录（如 asset/3d/animations）

动画配置

管理 3D 模型的动画列表。VRM 模型使用 .vrma 格式动画，PMX 模型使用 .vmd 格式动画。

字段说明
name动画名称，供状态映射引用
type动画类型：vrma（VRM Animation）或 vmd（VMD 格式）
file动画文件路径（相对于 animation_base_dir）
speed播放速度（默认 1）
fps帧率（默认 60），主要用于 VRMA 动画

从文件同步

编辑器提供两个同步功能，可以自动扫描 asset/3d/ 目录：

  • 从文件同步配置：自动检测 .vrm/.pmx 模型文件，填充 name、type、file 等模型配置
  • 从文件同步资源：扫描 .vrma/.vmd 动画文件，自动生成动画列表

新增同名状态

每个动画条目右侧的「新增同名状态」按钮可快速在 manifest 中创建对应的状态，并自动设置关联动画名。

导入文件夹

点击「📂 导入文件夹」按钮，可选择一个外部文件夹，将其内容递归复制到 Mod 的 asset/3d/ 目录下。适合从外部位置批量导入 3D 模型和动画资源。

💡 提示

3D 模型文件（.vrm/.pmx）和动画文件（.vrma/.vmd）需要手动放置到 asset/3d/ 目录下。PMX 模型的纹理文件放置到 asset/3d/textures/ 目录。

💡 支持的格式

VRM 模型搭配 VRMA 动画使用（通过 Three.js + @pixiv/three-vrm 渲染）；PMX 模型搭配 VMD 动画使用（通过 Three.js MMDLoader 渲染）。纹理支持 PNG、BMP、TGA、JPG、WebP。

📝 多语言文本

配置多语言的角色信息和对话内容。

角色信息 (info.json)

语言ID、语言名称、角色名称、角色描述

对话文本 (speech.json)

名称、内容(支持Markdown)、持续时间(秒)

支持占位符：{nickname}（昵称）、{days_used}（使用天数）、{usage_hours}/{total_usage_hours}（总使用时长，小时）、{uptime}（本次启动已运行时长，HH:MM:SS）

新增同名状态

每条对话文本右侧有「新增同名状态」按钮，点击后可快速在 manifest 中创建同名状态，并自动联动：

  • 关联文本：自动设为当前文本名称
  • 关联动画：若 Live2D 状态映射中存在同名条目，自动关联
  • 关联音频：若音频资源中存在同名条目，自动关联

Markdown 语法

  • **文本** → 加粗
  • [文本](链接) → 超链接
  • \n → 换行

语言管理操作

每种语言支持以下操作：

  • 复制语言：将整种语言的所有数据复制到剪切板
  • 粘贴语言：从剪切板粘贴语言数据，可新建语言或覆盖已有语言
  • 删除语言：删除指定语言（至少保留一种语言）

🔊 多语言音频

配置多语言的语音资源。音频文件支持 .wav 格式，需要手动放置到对应目录。

点击「一键导入」按钮可自动扫描音频目录中的 .wav 文件并批量导入为音频条目，省去逐个手动添加的步骤。

也可以点击「📥 导入」按钮单独导入一个音频文件。

💾 保存与导出

保存到文件夹

点击「💾 保存」（或「📦 导出」→「📂 到文件夹」），选择目标文件夹后写入。当前版本每次保存都会弹出文件夹选择器。

导出为 .tbuddy

点击「📦 导出」→「📦 到 .tbuddy」生成压缩包（包内会以 Mod ID 作为根目录）。

导出为 .sbuddy

当同目录下存在 sbuddy-crypto.exe 时，会显示「🔒 到 .sbuddy」按钮。点击后先导出 .tbuddy 再由外部工具转换为加密的 .sbuddy 格式。

保存/导出的文件

  • manifest.json
  • bubble_style.json（开启气泡样式时生成）
  • ai_tools.json（开启 AI 工具时生成）
  • asset/sequence.json, asset/img.json（序列帧 Mod）
  • asset/live2d.json（Live2D Mod）
  • asset/pngremix.json（PngRemix Mod）
  • asset/3d.json（3D Mod）
  • asset/live2d/ 下的所有文件（.model3.json、.moc3、贴图、.motion3.json、.exp3.json 等）将完整打包（Live2D Mod）
  • asset/*.pngRemix（PngRemix Mod）
  • asset/3d/（3D Mod）
  • text/[lang]/info.json, text/[lang]/speech.json（开启文本对话时生成）
  • audio/[lang]/speech.json（开启音频对话时生成）
  • preview.(png/jpg/jpeg/webp), icon.ico

❓ 常见问题

      
Q: 为什么无法打开文件夹？

      
A: 请使用 Chrome 86+ 或 Edge 86+ 浏览器。

      
Q: 动画不显示？

      
A: 检查图片是否在正确目录，路径和帧配置是否正确。

      
Q: 如何添加新语言？

      
A: 在「文本管理」或「音频管理」中点击「➕ 添加语言」。

      
Q: 预览图支持哪些格式？

      
A: 支持 PNG、JPG、JPEG、WebP 四种格式。

      
Q: 为什么提示 i18n 资源加载失败 / file:// 被拦截？

      
A: 请不要用 file:// 直接打开页面。双击 mod-tool/打开-mod编辑器.bat 启动本地服务器，再用浏览器访问 http://127.0.0.1:4174/。
